Вся цепочка генерации в одном месте: исходное изображение, получившийся MP4, animated WebP и промпт, который запустил движение.

Pipeline Как MP4 превращается в animated WebP ffprobe → ffmpeg → img2webp → webpmux
  1. 1. Входные данные Перед конвертацией через ffprobe считываются разрешение, длительность, FPS, количество кадров и видеотрек.
  2. 2. Декодирование видео ffmpeg -i input.mp4 -map 0:v:0 -fps_mode passthrough frame_%06d.png раскладывает видео в PNG-кадры без ресайза и без пересчёта FPS.
  3. 3. Тайминг анимации Длительность MP4 делится на количество кадров, а целочисленные задержки WebP распределяются так, чтобы сумма совпала с исходным видео. Для 30fps здесь используются задержки 33/34 ms.
  4. 4. Сборка animated WebP Кадры собираются через img2webp -loop 0 -sharp_yuv -lossy -q 95 -m 6 ... -o output.webp. Используется бесконечный loop, lossy WebP, качество 95 и максимальный режим сжатия.
  5. 5. Проверка результата webpmux и ffprobe используются для проверки разрешения, количества кадров, длительности, loop count и корректности WebP-контейнера.
